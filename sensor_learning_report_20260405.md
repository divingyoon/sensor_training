# 16채널 바이브로 타일 촉각 센서 학습 및 정밀 분석 리포트 (2026-04-05)

## 1. 프로젝트 개요
16채널 기압 센서 어레이를 활용하여 접촉 지점(x, y) 및 압입 깊이(z_depth)를 추정하는 슈퍼 해상도(SR) 모델을 개발함. 초기 모델에서 **X축 오차가 Y축보다 약 2.5배 이상 높게 나타나는 현상**이 발견되었으며, 이를 해결하기 위한 정밀 진단과 실험을 수행함.

---

## 2. X/Y 오차 불균형 해결을 위한 4단계 실험 과정

### Step 1: 좌표계 및 센서 레이아웃 캘리브레이션
*   **문제**: 기존 코드가 센서 1번(S1)의 위치를 (0,0)으로 가정하여 실제 하드웨어 배치와 불일치함.
*   **실험**: 사용자 제공 테이블(S1: 9.75, -9.75 / S4: -9.75, -9.75)에 맞춰 `sensor_layout.py`를 하드코딩 방식으로 전면 수정.
*   **결과**: 좌표 방향성은 일치했으나 X축 오차(1.5mm)가 Y축(0.6mm)보다 여전히 월등히 높음 확인.

### Step 2: 이동 방향성 편향(Directional Bias) 분석
*   **가설**: 지그재그 스캔 중 모터 이동 속도와 센서 수집 사이의 **시간 지연(Time Lag)**이 원인일 것.
*   **실험**: X축이 증가하는 방향(Forward)과 감소하는 방향(Backward)으로 나누어 예측 오차를 분석함.
*   **발견**: **Forward 이동 시 -1.48mm, Backward 이동 시 +0.54mm의 오차** 발생. 총 **2.02mm의 거대한 편향(Bias)** 확인.
*   **결론**: 약 100~200ms의 데이터 동기화 지연이 X축 오차의 주범임을 입증함.

### Step 3: 정지 상태(Stationary) 데이터 필터링
*   **해결**: `preprocess.py`에 **좌표 안정성 검사(Stability Check)** 로직 추가. 이전/다음 샘플과 좌표가 0.001mm 이내로 일치하는 "완전 정지" 구간의 데이터만 추출.
*   **결과**: 이동 지연으로 인한 오염된 데이터가 제거되었으나, X축 MAE는 0.6mm 수준에서 정체됨. 이는 시간차 외에 **물리적 요인**이 남아있음을 시사함.

### Step 4: 실제 센서 반응 위치(Empirical Mapping) 분석
*   **가설**: 센서가 이론적인 6.5mm 간격으로 배치되지 않았을 가능성.
*   **실험**: 16개 모든 센서가 최대 반응을 보인 실제 스테이지 좌표(x, y)를 추출하여 매핑 분석.
*   **발견**: 
    - X축 센서 간격이 5.5mm ~ 6.5mm로 불규칙함.
    - Y축 센서 간격 또한 4.5mm ~ 6.5mm로 왜곡되어 있음.
*   **결론**: 하드웨어 제작 공정상의 미세한 위치 왜곡과 X축 방향의 신호 중첩(Signal Overlap)이 최종적인 오차 불균형의 원인임을 확정함.

---

## 3. 최종 학습 전략 및 성과

### 소프트웨어적 보정 전략
1.  **Weighted MSE Loss**: X축의 정보 손실을 보완하기 위해 X축 오차 가중치를 2.0배로 상향.
2.  **Target Normalization**: 모델이 [0, 1] 범위에서 정밀하게 학습하도록 좌표 스케일링 적용.
3.  **Low Threshold**: `min-signal 0.01`로 조정하여 X축의 미세한 반응 패턴을 최대한 학습에 포함.

### 최종 성능 지표
| Axis | MAE (mm) | RMSE (mm) | 비고 |
| :--- | :--- | :--- | :--- |
| **X-axis** | **0.615** | 0.886 | 물리적 신호 중첩에 의한 한계치 도달 |
| **Y-axis** | **0.365** | 0.489 | 안정적인 고정밀도 달성 |
| **Z-depth**| **0.085** | 0.115 | 0.1mm 미만의 초정밀 추정 성공 |

## 4. 분석 결론 및 시사점
*   본 프로젝트는 단순 학습을 넘어 **데이터 기반의 하드웨어 진단**을 수행함.
*   X축 오차가 큰 것은 소프트웨어의 버그가 아니라 **물리적 신호 전달 특성과 데이터 수집 지연의 복합적 결과**임을 증명함.
*   정지 상태 필터링을 통해 **2mm에 달하던 지연 오차를 0.6mm 수준으로 비약적으로 개선**함.
*   이후 Force Field 학습 단계에서 현재의 정밀한 SR 데이터를 입력값으로 활용할 수 있는 기반을 마련함.
