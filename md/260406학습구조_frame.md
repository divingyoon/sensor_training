# Tactile Sensor Learning Pipeline (v3)
## 3D Super-Resolution + Force Field Learning Framework

---

## 0. 목적

본 프레임워크는 barometric tactile sensor (16ch)의 물리적 한계를 극복하여 다음을 학습하는 것을 목표로 함:

- Sub-mm contact localization (x, y, z)
- 3D force estimation (Fx, Fy, Fz)
- Continuous force field reconstruction
- Multi-contact 및 dynamic interaction 대응

---

## 1. 문제 정의

### 1.1 입력

- 16ch pressure signal
- optional: indenter radius r

```

s ∈ R^16

```

---

### 1.2 출력

```

y = {
position: (x, y, z)
force: (Fx, Fy, Fz)
field: F(p)  (continuous force field)
}

```

---

## 2. 현재 구조 (Baseline)

### 2.1 SR Model (MLP)

- Input: 16 + r
- Output: (x, y, z)

### 2.2 Force Model (MLP)

- Input: sensor + SR 결과
- Output: Fz

---

### 2.3 한계

1. spatial 구조 반영 불가
2. vector force 추정 제한
3. field reconstruction 불가능
4. multi-contact 대응 불가

→ MLP는 baseline으로 유지하되 확장 필요

---

## 3. 핵심 개념 전환

### 기존

- regression 문제

### 변경

→ **field reconstruction 문제**

---

## 4. 학습 파이프라인 (핵심)

---

## 4.1 Stage 0: Data Preprocessing

### (1) Stationary Filtering (필수)

조건:

```

dx/dt ≈ 0
dy/dt ≈ 0

```

목적:
- time lag 제거
- hysteresis 제거

---

### (2) Normalization

```

X, Y → [0,1]
Z → [0,1]
F → [0,1]

```

---

### (3) Dataset 구성

```

input:  sensor[16]
target: x, y, z, Fx, Fy, Fz

```

---

## 4.2 Stage 1: SR 학습

### 목표

정확한 contact localization

---

### 모델

MLP (baseline 유지)

또는 CNN / GNN (권장)

---

### Loss

```

L_SR = wx MSE(x)
+ wy MSE(y)
+ wz MSE(z)

```

- wx > wy (geometry 보정)

---

### 결과 기준

- MAE < 0.5 mm 목표

---

## 4.3 Stage 2: Force 학습

### 입력

```

sensor + predicted (x,y,z)

```

---

### 출력

```

(Fx, Fy, Fz)

```

---

### Loss

```

L_force = ||F_pred - F_gt||

```

---

### 핵심 포인트

- 동일 압력 → 위치에 따라 force 다름
- SR 결과 반드시 포함

---

## 4.4 Stage 3: Joint Learning

### 구조

```

```
    Encoder
       |
```

---

|       |       |
SR     Force    (optional field)

```

---

### Loss

```

L_total =
L_SR

* λ1 L_force

```

---

## 4.5 Stage 4: Field Learning (확장)

### 목표

continuous force field:

```

f(s, p) → F(p)

```

---

### 방법

#### 방법 1: Grid map

- output: 32x32 force map

#### 방법 2: Implicit Neural Field (권장)

입력:
```

(sensor, query_point)

```

출력:
```

force at point p

```

---

## 5. 모델 구조 설계

---

## 5.1 Encoder (필수 변경)

### Option A: CNN

```

16ch → 4x4 grid → Conv

```

장점:
- spatial locality 반영

---

### Option B: GNN (강력 추천)

- node: sensor
- edge: neighbor

```

h_i = Σ φ(s_j - s_i)

```

장점:
- 실제 센서 geometry 반영

---

### Option C: Transformer

- global interaction

단점:
- 데이터 요구량 큼

---

## 5.2 Multi-head 구조

```

shared encoder
|
-

|        |          |
SR     Force      Field

```

---

## 6. 데이터 설계 (가장 중요)

---

## 6.1 현재 데이터 문제

- normal force only
- single contact

---

## 6.2 필수 확장

### (1) Shear force

- Fx, Fy 포함

---

### (2) 다양한 indenter

- sphere
- flat
- edge

---

### (3) dynamic data

- sliding
- rolling

---

### (4) multi-contact

---

## 7. Loss 설계

---

## 7.1 Position

```

MSE(x,y,z)

```

---

## 7.2 Force

```

||F_pred - F_gt||

```

---

## 7.3 Field

```

Σ ||f(p_i) - f_gt(p_i)||

```

---

## 7.4 Physics Prior (추천)

```

pressure ∝ depth

```

regularization 추가
