# 16채널 촉각 어레이의 위치·수직힘 추론에서 축별 오차 지도가 달라지는 원인과 차세대 학습 구조 선택

## 요약

본 시스템은 6.5 mm 간격의 16채널 촉각 어레이 신호로부터 접촉 위치(x, y), 압입 깊이(z_depth), 수직힘(Fz)을 추정하는 문제이며, 현재 구현은 SR(위치·깊이)→FF(힘)로 직렬 연결된 2단 MLP 구조임. fileciteturn2file0

축별·출력별 오차 “핫스팟”이 서로 다른 위치에서 발생하는 현상은 비정상적인 예외가 아니라, (i) 세금(taxel) 수용장(receptive field)의 겹침이 위치에 따라 달라져 축별 민감도(∂signal/∂x, ∂signal/∂y, ∂signal/∂z)가 공간적으로 비등방(방향성)이며, (ii) 역문제(inversion)가 특정 방향에서 조건수가 나빠지는(ambiguity/ill-conditioning) 영역이 존재하고, (iii) 센서·구조물·데이터 로그의 비선형성/히스테리시스/라벨 오차가 출력별로 다르게 반영되기 때문임. citeturn4view0turn2view0turn7view0

모델 선택은 “입력 형태(16ch 정적 벡터 vs 시간 시퀀스)”, “실시간 요구(1 kHz급)”, “일반화 범위(인덴터/재질/멀티콘택트/동적 접촉)”에 따라 달라져야 함. 기본적으로 정적 단일 접촉·정지상태 데이터만 쓰는 현재 조건에서는 MLP가 성능/지연/구현 난이도에서 매우 강하며, 개선이 필요할 때는 (1) 좌표-인지형 CNN(4×4 재배열 + CoordConv) 또는 (2) 좌표 포함 Graph/Attention 인코더(GAT/Graph Transformer)로 “공간적 귀납편향”을 추가하는 접근이 비용 대비 효율이 큼. citeturn9search1turn2view0turn4view0

## 시스템 및 데이터 파이프라인 현황

현재 파이프라인은 두 개의 MLP가 직렬 연결된 형태임.

SR 모델은 입력 17차원(16채널 정규화 신호 + 인덴터 반지름 r)에서 (x, y, z_depth)를 회귀함. fileciteturn2file0  
FF 모델은 입력 20차원(16채널 + r + SR 출력(x, y, z_depth))에서 Fz를 회귀함. 즉, 힘 추정이 SR 오차에 구조적으로 종속되는(오차 전파) 형태임. fileciteturn2file0

데이터 측면에서 중요한 조치가 이미 포함되어 있음. 스테이지 좌표 갱신과 센서 샘플링 사이 약 200 ms 지연이 존재하며, 이동 중 데이터 학습 시 X축 이동 방향에 따른 히스테리시스/좌표 불일치(±1.01 mm급)가 발생하므로 정지 상태 데이터만 추출하는 전처리를 적용함. fileciteturn2file0 이는 “라벨-관측치의 1:1 대응”을 확보하는 핵심 선행 조건임.

타깃 정규화 범위(X,Y: [-9.75, 9.75], Z: [0, 1.5], Fz: [-1.26, 2.77])가 명시되어 있고, 체크포인트에 정규화 파라미터를 포함해 추론 시 물리 단위 환산을 일관되게 수행하도록 설계되어 있음. fileciteturn2file0

또한 X축 방향 정보 손실이 더 큰 배치 특성 때문에 손실함수에 X 가중치를 부여(Wx≈2~3)하여 X 오차 감소를 유도한 점이 기록되어 있음. fileciteturn2file0 이 조치는 “축별 오차 비대칭”을 줄이는 데는 유효하나, 아래에서 설명하듯 “공간적으로 국소적인 축별 조건수 문제”까지 자동으로 해소하지는 못함.

## 오차 맵에서 읽을 수 있는 현상

제공된 오차 히트맵 기준으로 다음 현상이 관찰됨.

x MAE는 특정 경계(특히 상단부)에서 상대적으로 커지는 띠·패치 구조가 나타나며, y MAE는 다른 위치(예: 특정 코너/점)에서 국소적으로 커지는 점상·패치 구조가 관찰됨. 이는 “동일 위치에서 x,y가 동시에 나빠지는” 패턴이 아니라 “축별로 다른 위치에서 나빠지는” 패턴이 명확히 존재함(사용자 질문 1의 핵심).  

z_depth MAE는 또 다른 위치에서 패치가 나타나며, Fz MAE는 x·y·z와 완전히 동일한 위치에만 의존하지 않고 별도의 “힘 오차 핫스팟”이 존재함. 특히 FF는 SR 출력(x,y,z_depth)을 입력으로 재사용하므로, SR의 국소 오차가 Fz 오차로 전파될 수 있으나, 동시에 “힘-변형 관계” 자체가 위치·재질·경계 조건에 의해 달라지는 비선형성 때문에 z 오차 지도와 1:1로 일치하지 않는 것이 자연스러움. fileciteturn2file0turn4view0turn5view0

(이미지 표시는 사용자 제공 자료 기반이며, 외부 문헌 인용 대상이 아님.)

## x·y·z·Fz 오차가 다른 위치에서 커지는 원인

본 절은 “왜 -9.75, 9.0 근처에서 x는 나쁜데 y는 그렇지 않은가”를 포함해, 축별/출력별 오차 핫스팟이 분리되는 원인을 **물리-계측-학습** 관점에서 분해함.

### 수용장 겹침 기반 SR에서의 방향별 관측가능성 차이

촉각 SR(물리 taxel 간격보다 더 미세한 위치 추정)은 “인접 세금들의 수용장이 겹치고, 그 겹침 패턴이 위치에 따라 달라지며, 다채널 신호를 종합해 가상 taxel/고해상 자극 지도를 복원한다”는 메커니즘에 기반함. citeturn4view0 즉, 동일한 위치 오차라도 x방향과 y방향으로 0.1 mm 이동했을 때 신호 변화량이 같을 이유가 없음.

이를 국소 선형화로 쓰면 다음과 같음.

- 관측: **s** ∈ ℝ¹⁶, 상태: **p** = (x, y, z)  
- **s** = f(**p**) + ε  
- 국소에서 Δ**s** ≈ J Δ**p**, J = ∂f/∂(x,y,z)

여기서 J의 열 벡터(∂**s**/∂x, ∂**s**/∂y)가 공간적으로 달라지며, 특정 위치에서는 ∂**s**/∂x가 작거나(민감도 부족), ∂**s**/∂x가 ∂**s**/∂z와 거의 공선(collinear)처럼 되어(분리 불가) x가 더 “모호해지는” 영역이 생김. 반대로 y는 충분히 분리 가능하면 y 오차는 작게 유지됨. 이런 “본질적 불확실성/모호성”은 taxel 기반 촉각 신호에서 중요한 이슈로 반복적으로 지적되며, 이를 줄이기 위해 다중 접촉 분포나 기하 정보를 더 적극적으로 쓰는 방법이 제안됨. citeturn2view0turn4view0

사용자 예시(-9.75, 9.0 부근)는 사실상 경계에 가까운 영역이며, 경계에서는 활성화되는 세금 조합이 한쪽으로 치우치고 수용장 겹침의 기하가 비대칭이 되기 쉬움. 따라서 x방향으로 움직일 때와 y방향으로 움직일 때의 신호 변화율이 달라져 “x만 유독 나쁜 구간”이 생기는 것이 이론적으로 자연스러움. citeturn4view0turn3search3

### 센서 채널 편차·국소 결함이 축별 오차로 나타나는 구조

16채널에서 x 위치는 대체로 “좌↔우 센서들의 상대적 크기/기울기 정보”에 더 민감하고, y 위치는 “상↔하” 정보에 더 민감한 경우가 많음(물리 배치가 격자형일 때 특히 그러함). 이때 특정 채널(또는 특정 열/행)에 드리프트·노이즈·감도 저하가 있으면 x 또는 y 중 하나에 더 큰 타격을 주는 “축 선택적 취약점”이 발생함.

사용자 보고서에서도 히트맵으로 손상/변화 지점을 식별하는 도구를 언급하고 있어, “국소 채널 문제 → 국소 축별 오차 핫스팟” 가설이 실무적으로 유력함. fileciteturn2file0

또한 소프트/폴리머 기반 촉각 센서는 비선형성과 히스테리시스를 내재하며, 이는 전통적 보정으로 해결이 어렵다고 보고되어 왔음. citeturn7view0 이 비선형성은 위치·깊이·속도(동적 접촉) 조건에 따라 다르게 나타날 수 있어, 결과적으로 축별 오차 지도가 달라지는 방향으로 작동하기 쉬움.

### 라벨 정합(좌표계·타임베이스) 불일치의 잔여분이 축별로 남는 경우

보고서에서 200 ms 지연과 이동 중 학습 시 ±1.01 mm 수준의 불일치를 확인하고 정지상태 필터링을 적용했다고 명시되어 있음. fileciteturn2file0 그럼에도 다음 잔여 요인이 남을 수 있음.

- 스테이지 축별 백래시/미세 비선형 보정 오차가 x와 y에 비대칭으로 존재하는 경우(라벨 자체가 축별로 다른 분산을 가짐)  
- “정지 상태” 판정이 dx/dt, dy/dt만으로 정의되어 있을 때, 실제로는 z 압입 과정에서 미세한 측면 미끄럼/탄성 복원(특히 경계) 등이 섞이는 경우  
- 센서 반응이 완전히 정적이 아니라 점탄성 완화(relaxation)를 포함할 때, 샘플링 타이밍이 축별·영역별로 다르게 영향을 주는 경우(예: 특정 위치에서 접촉 면적이 커서 완화가 더 큼)

이들은 오차 지도를 “위치별로, 그리고 축별로” 다르게 만들 수 있으며, 특히 경계/코너에서 더 두드러지기 쉬움. citeturn7view0turn4view0turn2file0

### z와 Fz 오차 지도가 서로 다를 수밖에 없는 이유

z_depth는 “변형량(또는 압입 깊이)”에 가까운 기하 변수이고, Fz는 “접촉 반력”으로 재질 강성, 경계 조건, 접촉 면적, 마찰/전단 상태에 의해 달라지는 역학 변수임. 따라서 동일 위치라 하더라도 z와 Fz의 관계가 단순 단조/선형일 필요가 없으며, 다른 위치에서는 동일 z에서도 Fz가 달라질 수 있음(예: 경계가 더 유연하면 같은 z에서 반력이 작아짐). 이는 촉각에서 위치 추정과 힘 분포/힘 추정이 함께 다뤄지는 이유이기도 함. citeturn4view0turn5view0turn3search3

또한 현재 FF는 SR 출력(x,y,z_depth)을 입력으로 사용하므로, SR 오차는 Fz 오차에 직접 주입됨. fileciteturn2file0 따라서 “z는 괜찮은데 Fz가 나쁜 위치”, 또는 “z는 나쁜데 Fz는 덜 나쁜 위치” 모두 구조적으로 가능함(FF가 신호 크기에서 Fz를 더 직접적으로 읽는 경우 등).

## 원인 분리를 위한 진단 실험 설계

아래 진단은 “왜 x는 여기서만 나쁜가”를 가설 수준이 아니라 데이터로 분리하기 위한 최소 세트임. (추가 데이터 수집 없이도 로그로 수행 가능한 항목 중심으로 구성함.)

첫째, **국소 민감도 지도(Jacobian proxy) 계산**이 필요함. 각 그리드 포인트에서 (x±δ, y), (x, y±δ) 인접 샘플을 이용해 ‖Δs‖/δ를 계산하면, 위치별로 “x로 움직일 때 신호가 얼마나 변하는가”와 “y로 움직일 때 얼마나 변하는가”를 직접 지도화할 수 있음. 이 값의 비율이 큰 곳에서 해당 축 오차가 커지는지 검증하면, “근본 관측가능성 부족”인지 빠르게 판별 가능함. 수용장 겹침 기반 SR이 위치 민감도를 제공한다는 논리와 일치하는 테스트임. citeturn4view0turn3search3

둘째, **채널 기여도/결함 탐지(채널 ablation)**가 필요함. 16채널 중 하나씩 마스킹(또는 노이즈 주입)하여 x,y,z,Fz 성능 저하를 측정하면, 특정 채널이 x에만 과도하게 중요하거나 특정 영역에서만 문제를 만드는지를 확인 가능함. 보고서에서도 히트맵 기반 품질 진단 도구를 언급하고 있으므로, 이 검증은 현재 워크플로에 자연스럽게 연결됨. fileciteturn2file0

셋째, **좌표계 정합(affine calibration) 잔차 분석**이 필요함. 예측 (x̂, ŷ)와 GT(x, y) 사이에 최적 affine 변환을 한 번 맞춘 뒤(회전/스케일/전단 포함), 잔차 지도가 크게 줄어들면 “모델 문제가 아니라 좌표계/테스트베드-센서 매칭 문제” 비중이 큼. 반대로 잔차가 그대로면 “센서/모델의 국소 조건수 문제” 비중이 큼.

넷째, **Fz 오차 핫스팟의 원인 분해**가 필요함. FF 입력이 SR 출력에 의존하므로, (i) FF를 GT(x,y,z)로 구동했을 때의 Fz 오차 지도와, (ii) SR 출력으로 구동했을 때의 Fz 오차 지도를 비교하면 “FF 자체 한계”와 “SR 오차 전파”를 분리할 수 있음. 현재 FF 구조 정의상 가능한 실험임. fileciteturn2file0

## 아키텍쳐 선택: MLP, CNN, Temporal, Attention/GNN

본 절은 사용자 질문 2(“CNN, CNN+LSTM, Transformer 중 무엇을 택해야 하는가”)에 대해, **데이터 형태와 목표 기능**에 따라 선택 기준을 명확히 제시함.

### MLP의 한계와 강점의 경계

입력이 16채널(낮은 차원)이고, 정지 상태 단일 접촉·단일 인덴터(또는 제한된 인덴터 반지름)라면 MLP는 표현력 부족이 아니라 “데이터 정합/물리적 모호성”이 병목이 되는 경우가 많음. 실제로 보고된 성능도 SR Total MAE 0.338 mm, Fz MAE 0.602 N 수준으로 이미 상당히 높음. fileciteturn2file0

MLP의 실질적 한계는 주로 아래 조건에서 나타남.

- 멀티콘택트/접촉 패치(면)까지 다루려는 경우(출력이 단일 (x,y)로 충분하지 않음)  
- 동적 접촉(압입/해제, 미끄럼)에서 시간적 이력(히스테리시스/점탄성)이 중요해지는 경우 citeturn7view0turn6view0  
- 센서 레이아웃/형상이 바뀌거나, taxel 좌표가 비정규/곡면으로 확장되는 경우(벡터 MLP가 구조 일반화에 불리) citeturn2view0

### CNN이 유리한 조건

16채널을 4×4로 재배열하면 “세금 공간 구조”를 활용할 수 있고, 작은 CNN은 MLP 대비 파라미터 증가 없이도 국소 패턴(주변 세금의 상대적 변화)을 더 잘 포착하는 경우가 있음. 또한 “좌표를 추가 채널로 주입(CoordConv)”하면 위치 회귀가 요구하는 translation dependence(좌표 변환 문제)를 더 직접적으로 해결할 수 있음. citeturn9search1turn9search4

촉각에서 접촉 위치와 힘 분포/힘을 동시에 추정하기 위해 CNN 기반 회귀를 사용하는 흐름은 이미 확립되어 있으며(특히 비전 기반 촉각에서), X/Y/Z 방향 변위·힘 분포를 회귀하는 데이터 수집/학습 파이프라인이 제시된 바 있음. citeturn5view0  
세금 기반에도 “촉각 신호를 2D 이미지로 해석”하여 SRGAN 등으로 고해상 표현을 만드는 접근이 보고되어 있음. citeturn2view0turn3search2

### CNN+LSTM(또는 ConvLSTM)이 유리한 조건

데이터 수집이 “압입 과정의 시계열”을 포함하고, 그 이력이 히스테리시스/점탄성 때문에 예측에 필요하다면 시간 모델이 필요함. 소프트 촉각 센서에서 히스테리시스/비선형이 보정 난점이며, RNN류를 포함한 ML 접근이 이를 다루기 위해 사용되어 왔음. citeturn7view0  
또한 촉각 시계열에서 공간+시간 특징을 동시에 학습하기 위해 ConvLSTM을 사용하는 사례가 있으며, 짧은 시간 창(예: 수십 ms)에서도 유의미한 spatio-temporal 특징을 학습한다고 보고됨. citeturn6view0

다만 현재 파이프라인은 “정지 상태 데이터만 100% 추출”하는 방식이라, 의도적으로 시간 정보를 제거한 설정임. fileciteturn2file0 이 경우 CNN+LSTM을 도입해도 이득이 제한적일 수 있으며, 오히려 데이터/라벨 정합과 공간적 조건수 문제를 먼저 해결하는 것이 우선순위가 됨.

### Transformer/Attention이 유리한 조건

세금 수가 늘거나(예: 32~256), 멀티콘택트에서 “어떤 세금 조합을 어느 정도 참고할지”가 상황별로 달라질 때 attention은 정보 집계를 유연하게 함. 실제로 sparse taxel로부터 고해상 자극 분포를 추정하는 프레임워크에서 self-attention을 결합한 SR 모델이 제안되어 있으며, 수용장 겹침을 활용해 가상 taxel을 생성하고 위치 오차를 줄이는 결과가 보고됨. citeturn4view0

또한 taxel 신호의 비정형성(레이아웃 다양성)을 다루기 위해 graph 표현 + GNN 인코더를 사용하는 흐름이 등장했으며, taxel 배열을 그래프로 표현하면 공간 배치 자체(좌표)와 비접촉 상태를 함께 인코딩할 수 있다고 명시됨. citeturn2view0  
사용자 시스템이 향후 곡면 손가락/다른 배열로 확장되는 로드맵이라면, “처음부터 그래프/좌표 기반”으로 가는 편이 재사용성이 높음.

## 권장 최종 구조와 학습·평가 프로토콜

### 권장 구조

요구 기능이 (x,y) + Fz 두 가지이며, 현재처럼 단일 접촉 정적 셋업을 유지한다는 전제에서 다음 우선순위를 권장함.

첫째, **단일 인코더 + 멀티헤드 멀티태스크**가 2단 직렬(SR→FF)보다 안정적인 경우가 많음. 직렬 구조는 SR 오차가 FF로 전파되는 구조적 문제가 있기 때문임. fileciteturn2file0 다만 멀티태스크는 손실 가중치에 민감하며, 서로 다른 단위/스케일을 동시에 학습할 때 가중치 튜닝이 병목이 될 수 있음. 이를 완화하기 위한 불확실도 기반 자동 가중치(learned homoscedastic uncertainty) 접근이 널리 사용됨. citeturn9search0turn9search3

둘째, 인코더는 아래 중 하나를 선택하는 것을 권장함.

- **CoordConv 기반 소형 CNN**: 입력을 4×4로 reshape하고, (x좌표 채널, y좌표 채널)을 추가하여 위치 회귀 안정성을 높이는 방식임. CoordConv는 conv에 좌표 채널을 추가해 좌표 변환 문제를 직접 다룬다는 점이 핵심임. citeturn9search1turn9search4  
- **좌표 포함 Graph Attention(GAT) / Graph Transformer**: 노드=taxel, 노드 특성=[신호, (x_i,y_i)]로 두고 message passing/attention으로 집계하는 방식임. taxel 신호를 그래프로 표현하고 GNN으로 처리하는 접근이 “레이아웃 불변/일반화”를 목표로 제안된 바 있음. citeturn2view0

현재 16채널/격자 배열이면 CNN이 구현 난이도와 실시간성 측면에서 유리하며, 배열이 비정형이거나 곡면으로 확장되면 그래프가 유리함.

셋째, 시간 모델은 조건부로 도입함. 정지상태만 쓰는 현재 설정에서는 시간 모델의 이득이 제한적임. 반면 “압입-유지-해제”의 전체 이력을 포함시키거나 동적 이벤트(미끄럼/충격)를 추론해야 한다면 TCN 또는 ConvLSTM을 얹는 구성이 강력함. 촉각 시퀀스에서 ConvLSTM으로 시공간 특징을 학습하는 효용이 보고되어 있음. citeturn6view0turn8search6

### 학습·평가 프로토콜

오차 핫스팟 문제는 모델 구조 변경보다 “평가 설계”가 원인을 드러내는 경우가 많으므로 다음을 권장함.

- **그리드 홀드아웃 평가**: 특정 위치 블록(예: 3×3 또는 5×5 패치)을 통째로 테스트로 제외하고 학습하여, 모델이 “보간(interpolation)”만 하는지 “진짜 일반화”를 하는지 평가해야 함. taxel SR 논문들이 데이터 품질과 일반화 문제를 반복적으로 강조하는 맥락과 일치함. citeturn4view0turn2view0  
- **Fz 분해 평가**: 앞서 제시한 “GT(x,y,z)로 FF 구동 vs SR(x̂,ŷ,ẑ)로 FF 구동” 비교로 SR→FF 오차 전파량을 정량화해야 함. fileciteturn2file0  
- **축별 민감도-오차 상관 분석**: Jacobian proxy 지도와 x/y 오차 지도의 상관을 보고, 센서 물리/배치 기반의 관측가능성 문제가 맞는지 결론을 내려야 함. citeturn4view0turn3search3  
- **채널 수준 품질 게이팅**: 채널 ablation으로 의심 채널이 확인되면, 해당 채널을 보정/교체하거나 학습 시 채널 드롭아웃/노이즈 주입으로 강건성을 확보해야 함(보고서의 품질 진단 도구 언급과 정합됨). fileciteturn2file0

이 프로토콜을 통과한 뒤에도 “특정 위치에서만 x가 나쁜” 현상이 남는다면, 이는 모델 부족이 아니라 센서-구조물-경계 조건이 만드는 근본적 모호성(조건수 문제)일 가능성이 커지며, 그때는 (i) taxel 배치/커버층 두께/재질로 receptive field를 재설계하거나, (ii) 멀티탭·멀티접촉 같은 추가 관측으로 불확실성을 줄이는 방향이 정답에 가까워짐. citeturn4view0turn2view0turn3search2
