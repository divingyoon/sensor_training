## 1. 3-axis Force Sensing

Sensing Point $P_i$는 아래와 같이 구성됨.

$$
P_1=(\frac{s_{1,x}+s_{2,x}+s_{5,x}+s_{6,x}}{4},\frac{s_{1,y}+s_{2,y}+s_{5,y}+s_{6,y}}{4}) \\ 
P_2=\dots
$$

즉, 센서 4개의 중심이 P의 (x,y)의 좌표가 됨.

각 P점과 연관된 4개의 센서들의 출력 관계를 통하여 $F_x, F_y,F_z$를 구할 수 있음.

$$
\tilde{F}=\begin{bmatrix}
F_{p_1,x}& F_{p_2,x} & \dots & F_{p_9,z} \\ 
F_{p_1,y}& F_{p_2,y} & \dots & F_{p_9,z} \\
F_{p_1,x}& F_{p_2,z} &\dots & F_{p_9,z}
\end{bmatrix}

=
\tilde{C} (\tilde{S} - \tilde{\rm O}) \\
=
\begin{bmatrix}
c_{1,1} & c_{1,2} &\dots  & c_{1,15} &c_{1,16} \\ 
c_{2,1} & c_{2,2} &\dots & c_{2,15} &c_{2,16}\\ 

c_{3,1} & c_{3,2} &\dots & c_{3,15} &c_{3,16}
\end{bmatrix}
(\begin{bmatrix}
s_1 \\ 
s_2\\ \dots \\
s_{15} \\
s_{16}
\end{bmatrix}
- \tilde{\rm O}))
$$

$\tilde{C}$ : Calibration Matrix

$\tilde{S}$ : Sensor Raw Count Matrix

$\tilde{\rm O}$ : Baseline 

Commercial sensor( AIDIN- AFD50)센서를 사용하여 $F_x, F_y,F_z$와 $(s_1, s_2, \dots, s_{15}, s_{16})$을 기록하여 Calibration Matrix를 구함.

구한 Calibration Matrix 를 이용하여 역으로 $F_x, F_y,F_z$를 계산함.

예를 들어, $P_1$ 기준

$$
F_{x,\rm P_1}=c_{1,1}S_1+c_{1,2}S_2+c_{1,5}S_5+c_{1,6}S_6 \\
F_{y,\rm P_1}=c_{2,1}S_1+c_{2,2}S_2+c_{2,5}S_5+c_{2,6}S_6 \\
F_{x,\rm P_1}=c_{3,1}S_1+c_{3,2}S_2+c_{3,5}S_5+c_{3,6}S_6 \\
(S_i  \approx 0 \ \ ;\ i\neq1,2,5,6)
$$

## 2. Contact Force and Position Sensing

$(s_1, s_2, \dots, s_{15}, s_{16})$의 출력을 관계를 학습하여 외력(또는 접촉물)에 대한 접촉 위치 $(x,y,z,F_z)$ 및 분포를 를 출력함. 

즉, 특정한 위치를 출력하거나 접촉된 영역과 응력을 센싱하는 것.

## 3. Slip-Detection

물체가 미끄러질 때 발생하는 진동을 감지하여 미끄러짐 발생 여부에 대한 센싱을 할 수 있음.

$(s_1, s_2, \dots, s_{15}, s_{16})$에서 출력되는 Raw Count 값을 DWF 변환을 통해 특정 주파수에서 일어나는 Slip 주파수를 저장함.

추후 미끄러짐이 발생되었을 때의 진동이 센서에게서 감지되면 슬립 신호를 출력함.

# How to calibrate and acquire data

## 1. Data Table

![image.png](image%202.png)

Time : 데이터들을 시계열을 맞추기 위한 열

Indenter : 일반적으로 구 형태를 사용하므로 **구의 지름**을 기입할 예정.

P_i : 3-axis 용 Sensing Point Labeling 1~9로 표현. 0인 경우는 3축용 센싱 포인트가 아님. 

$x,y,z$ : 센서의 기준점으로부터의 위치 정보. Z는 참고로 센서가 눌리는 Depth랑 같음.

$\theta$ : 센서의 x축 또는 y축을 기준으로 회전한 각도.

$F_x, F_y,F_z$ : AFD50 센서에서 기록되는 힘 정보

$(s_1, s_2, \dots, s_{15}, s_{16})$ : BMP384에서 출력 되는 Raw count

## 2. 3-axis Force Sensing

1. 특정 Sensing Point $P_i$ 기준으로 testbed를 이동.
2. 이후 실험을 통해 time,x,y,z, $F_x, F_y,F_z$ ,$(s_1, s_2, \dots, s_{15}, s_{16})$ 값을 기록함. 
    1. 기록한 데이터의 구성은 단일 축에 대한 정보를 위한 데이터와 복합 하중을 위한 데이터 모두를 포함.
    2. $P_i$에 대한 정보는 x,y 좌표로 나타내거나 또는 x,y 열을 없애고 $P_i$열들을 추가하여( 1 또는 0으로 표기) 어느 점에서 진행한 것인지 표기.
3. Calibration Matrix를 계산
4. Commercial Sensor와 함께 데이터 평가

## 3. Contact Force and Position Sensing

1. 데이터 테이블에 indenter 열을 추가 → 구 모양이고, 지름으로 표기 
2. 실험을 통해 time, indenter, x, y, z, $F_z$, $(s_1, s_2, \dots, s_{15}, s_{16})$ 값을 기록함.
    1. 모든 센서 및 textbed 모터 등 초기화 후 indenter 기입.
    2. 현 위치(시작 지점은 (0,0))에서 z의 임계값 또는 센서의 출력이 ‘0’이 될 때까지 $F_z$를 기록함.
    3. 특정 x 단위로 모터 이동, 예를 들어 (1,0)으로 이동후 b를 반복함.
    4. 임계 x값(18.5mm 등)에 도착하면 x는 0으로 돌아오고, 특정 단위로 y를 이동시킨 후 위 과정을 반복. 
    5. 시작 지점부터 $(x_{\rm threshold},y_{\rm threshold})$에 도착할 때까지 위 과정을 반복 후 종료.
    6. indenter를 바꿔가며 다양한 데이터를 취합
3. 취합된 데이터를 input($(s_1, s_2, \dots, s_{15}, s_{16})$ 과 Label $(x,y,z,F_z)$을 구분하고, Machine Learning의 방법으로 input에 대한 output 학습을 진행함.
4. 최종 학습 output은 아래와 같고, 학습하지 않았던 다양한 크기의 indentor를 가지고 평가함.
    
    1) 특정 위치에서의 접촉 위치와 힘 (super-resolution)
    2) 접촉 면적과 응력분포 이미지 추론
    

## 4. Slip-Detection

1. 일정한 indenter로 센서를 누르고 있는 상태($F_z$존재)로  x 또는 y 방향으로 testbedt를 이동시킴
2.  time, indenter, x, y, z, $F_z$, $(s_1, s_2, \dots, s_{15}, s_{16})$를 기록
3. slip이 일어난 시점을 역산
4. 그 전후로의 센서값들을 DWF 변환을 통하여 SLIP 발생 시의 주파수를 추출함.
5. 이러한 주파수가 센싱 도중에 측정되면 SLIP이 발생했다는 정보를 출력함.

## 1. Calibration Matirx을 계산할 때를 보면, 차원이 맞지 않음. (3x1) = (3x4) (4x1) 인 상황에서 (3x4) 보정행렬을 구할 수 있는가?

- 챗지피티 답변 : 가능
    
    입력 차원이 출력보다 크기 때문에 모든 힘 방향을 정확히 구별하기는 어려울 수 있습니다. 그러나 **의사역행렬(pseudo-inverse)**이나 최소자승법을 이용하여 주어진 데이터에 맞춰 변환 행렬을 구할 수 있습니다.
    
    입력이 4차원이고 출력이 3차원이어도, 충분한 교정 데이터를 확보하면 최소자승 형태로 C를 구해 힘을 재구성할 수 있습니다. 다만 센서 사이 감도 차이나 제작 오차로 행렬 요소가 이상적 이론값과 달라질 수 있으므로 , 각 채널별 감도 보정(감도 편차 보상)과 교정 실험이 중요합니다. 결론적으로, 4×1 센서 값으로 3×1 힘 벡터를 추정하는 3×4 보정 행렬을 **의사역행렬** 기반으로 구하는 것은 가능하며, 실제 사례에서도 낮은 오차로 힘 측정이 검증되었습니다

calibration 과정에서 문제가 없을 것으로 보이는가? 문제 또는 고려되는 문제들은 무엇인가?

## 2.2 calibration matrix를 구하고 나서도, 3축 센싱 등의 기능을 구현할 때, 제작 센서의 현재 구현 시점에서, 초기화를 시작할때 bmp384 각각의 raw값을 취득하여 정규화를 시킨 후에 기능을 구현(3축 센싱, 접촉위치, 슬립 등)하려고 한다.
논리 및 기능적으로 문제가 없는가? 문제가 있다면 어떻게 해결할 수 있는가?

- 챗지피티 답변 : 비추천
    
    세션마다 $\frac{x_{max} - x_{in}}{x_{max}}$ 로 센서 출력을 정규화하는 방식을 쓰면 각 측정값을 0~1 범위로 맞추게 됩니다. 하지만 이렇게 처리하면 보정 행렬이 기대하는 **절대값 스케일** 이 변형되므로, Calibration 행렬의 의미가 달라질 수 있습니다. 즉, 정규화는 센서 간 편차나 드리프트를 일부 상쇄할 수는 있으나, 힘-압력 관계가 선형이면 원래 선형 계수를 변경해 버려 측정 정확도가 떨어질 위험이 있습니다. 실제로 De Clercq 등(2024)은 **교정 없이** 압력 분포를 추정할 수 있는 방법을 제안했지만, 보정이 적용되면 오차(MAE)가 1.83N에서 1.02N으로 크게 개선됨을 보여주었습니다 [biblio.ugent.be](https://biblio.ugent.be/publication/01J2ZZWZZA9DXS7N0CMAGJ8JN9/file/01J93XD02N3634HJBCZ7P80VA6.pdf#:~:text=wide%20range%20of%20indenters,02%20N%20with%20calibration)  이는 적절한 보정 과정을 거치는 것이 성능 향상에 중요함을 보여줍니다. 따라서 매 세션마다 단순 정규화하는 방법은 Calibration **매트릭스가 학습된 스케일을 일관되게 유지하지 못해 기능에 부정적인 영향**을 줄 수 있습니다. 권장하는 접근법은 센서별 영점(offset)과 감도(sensitivity)를 개별 측정하여 보정한 후, **원시(raw)** 데이터에 보정 행렬이나 ML 모델을 적용하는 것입니다. 실제 시스템에서는 **센서 바이어스 제거 및 공통 기준(force sensor)과의 동기 캘리브레이션이 필요**합니다.
    

## 3. 3-axis force sensing을 calibration matrix를 사용하지 않고, 기계학습 또는 딥러닝 등으로도 구현할 수 있는가?
1) 가능하다면 calibration matrix과 추천하는 학습 방법을 비교해서 각각의 장단점이 무엇인가?
2) 학습이 가능하다면 학습을 하려면 어떤 방식을 추천하는 가?
3) 학습을 하기 위해서 어떤 식으로 준비해야하는가?

- 챗지피티 답변 : 가능
    
    충분한 크기로 준비한 힘-센서 데이터셋을 바탕으로 **다층 퍼셉트론(MLP), 지원벡터회귀(SVR), 랜덤포레스트나 신경망(딥러닝) 회귀 모델**을 학습할 수 있습니다. 
    
    준비 단계로는 **다양한 크기·방향의 힘을 주고 센서 출력을 수집하여 학습 레이블(3축 힘)을 확보**해야 합니다. 필요 시 세 축 힘과 비례하되 비선형성이 있는 관계를 커버하기 위해 **비선형 활성화 함수를 갖는 신경망 사용이 유리**할 수 있습니다. **ML 모델 학습 전에는 각 센서의 오프셋 제거와 스케일 조정, 노이즈 제거(필터링)가 중요**하며, 훈**련 후 검증 데이터에서 오버피팅 여부를 확인**해야 합니다. 물론 계산 복잡도를 줄이기 위해 간단한 회귀나 가우시안 프로세스를 시도해 볼 수도 있습니다. 하지만 Guided 디자인 이론이나 최근 연구들에 따르면, ML 기반 추정은 복잡한 비선형 관계를 모델링하고 소수 센서를 이용해 더 높은 정확도를 얻는 데 효과적일 수 있습니다
    

## 4.  contact positon sensing 관련
1) 어떤 방식의 학습을 추천하는 가?
2) 현재 내가 추출하는 data table 자료로 충분한가? 충분하지 않다면 무엇을 더 추가해야 하는가?
3) contact position 접촉 위치(응력 분포 등)를 실시간으로 표현하기 위한 그림을 만들려면 어떻게 해야 하는가?

- 챗지피티 답변 : 가능
    
    접촉 위치(텐서의 면 상 접촉점 좌표) 추정에도 머신러닝이 유용합니다. 예를 들어, 4×4 배열 센서의 16개 출력을 입력으로 받아 접촉점(x,y)을 회귀하는 모델을 들 수 있습니다. CNN, MLP, 또는 k-NN/서포트벡터머신 같은 모델이 후보가 됩니다.
    
    특히 Hou 등(2024)은 바로메터 기반 센서의 위치 추정에 CNN 계열 네트워크를 사용하여, 센서 간격 6mm를 0.284mm 급으로 높은 유효 해상도로 개선했고, 강체 객체 방향 분류에는 CNN-LSTM 모델로 평균 86.9% 정확도를 달성했습니다 [researchgate.net](https://www.researchgate.net/publication/386261451_Location_and_Orientation_Super-Resolution_Sensing_With_a_Cost-Efficient_and_Repairable_Barometric_Tactile_Sensor#:~:text=Machine%20learning%20techniques%20are%20applied,orientations%20is%20split%20into%204)
    
    **CNN**을 통해 공간적 센서 패턴을 학습하면 접촉 위치와 객체 방향 분류 모두 가능함을 보여주었습니다. (LSTM은 연속 신호 처리에 용이하므로, 접촉 지점의 추적 변화나 시퀀스 데이터를 사용할 때 유리합니다.)
    
    ---
    
    현재의 데이터 수집 방식(압력을 걸어주는 힘 센서와 동기화된 센서 출력 기록)이 충분한지는 **보정 데이터 품질**에 달려 있습니다. **접촉 위치 학습에는 실제 접촉 위치를 정확히 알 수 있는 기준(예: 좌표계 또는 객체 격자)에서 다양한 점을 눌러가며 데이터를 모으는 것이 좋습니다.** 현재 방식이 임의의 접촉 위치를 충분히 대표하도록 커버한다면 괜찮지만, 편향이 있을 경우 **추가 데이터(예: 다양한 위치/각도/소재) 확보가 필요**합니다.
    
    접촉 시 실시간 압력 분포를 시각화할 때는 일반적으로 *2D 컬러맵*이나 *3D 표면 플롯* 형태를 사용합니다. 예를 들어, 실제 촉각 패드 위 각 센서의 압력값을 컬러강도나 높이로 표현하여 현재 압력 지도를 보여줄 수 있습니다. 위 그림은 **상용 압력 매핑 소프트웨어(Chameleon)를 이용해 16개 센서 출력을 2D/3D 그래프로 실시간 시각화한 예시**입니다. 이처럼 센서 위치를 실제 지오메트리에 대응시켜 2D 히트맵(컬러맵)이나 3D 메쉬로 렌더링하면, 접촉 압력의 분포와 강도를 직관적으로 확인할 수 있습니다. (소프트웨어적으로는 각 센서 출력을 그리드 상 좌표에 맵핑한 뒤, 색상이나 높낮이로 나타내는 방식이 일반적입니다.)
    

## 5. 3-axis force sensing은 전체 센서에서 9개의 센싱 포인트에서만 추출하고 있다. 접촉위치와 3축 센싱 부분을 연관 지어서 표현할 수 있는 방법이 없나? 
즉, 접촉면적분포를 보여주면서도, 접촉면적에서 센싱 포인트가 같이 표현되어서 접촉물이 센서에 어떻게 닿고 있고, 어느 방향으로 힘을 가하면서 접촉되어있는 지 알고 싶다. 가능한가? 가능하다면 어떻게 해야하는 가?

- 챗지피티 답변 : 가능
    
    3축 힘(9개 이산 포인트) 측정과 접촉 분포를 결합하여 하나의 표현으로 통합하는 방법도 가능합니다. 한 가지 접근은 **압력 분포 재구성** (pressure reconstruction)을 통해 물체-센서 간 접촉 면 전체의 압력 맵을 구하고, 이로부터 총합 힘(3축 합력)과 접촉 중심(Center of Pressure)을 계산하는 것입니다. 예를 들어 De Clercq 등(2024)의 soft barometric 센서는 **파라메트릭 압력 분포 모델** 을 반복적으로 적합(fitting)하여 접촉 패턴 전체를 재구성하였고, 이를 통해 총 접촉력(normal force)과 접촉 개수까지 추론했습니다. 논문에서는 “감압 재구성을 통해 정상(contact count), 정상력(normal force) 및 방향 변화를 추정할 수 있다”고 기술했으며, 보정 없이 힘을 추정해 MAE 1.83N, 보정 후 1.02N의 정밀도를 얻었 [biblio.ugent.be](https://biblio.ugent.be/publication/01J2ZZWZZA9DXS7N0CMAGJ8JN9/file/01J93XD02N3634HJBCZ7P80VA6.pdf#:~:text=The%20presented%20work%20introduces%20a,iterative%20process%20is%20implemented%20to)따라서 센서값으로부터 압력 분포를 추정하면, 그 위에서 **접촉력의 합과 중심점** (center of pressure) 계산을 같이 수행할 수 있어, 힘과 분포 정보를 동시에 표현할 수 있습니다.
    
    시각적으로는 압력 분포 맵 위에 힘 벡터나 힘 크기 정보를 중첩하는 방법이 있습니다. 예를 들어, 압력 분포 히트맵을 배경으로 센서판 중심에서 외부로 향하는 화살표 벡터로 순힘(resultant force) 방향과 크기를 표시하거나, 압력값의 총합을 그래픽으로 나타낼 수 있습니다. 실제 로봇 애플리케이션에서는 이런 압력 맵과 힘벡터를 GUI나 HUD에 동시에 그려주어 물체가 센서표면에 어떻게 닿고 밀리는지 한눈에 볼 수 있습니다. 요약하면, **정상 및 전단 성분까지 포괄하는 총합력**과 **접촉 패드 전체 압력 분포**를 동시에 표시함으로써, 접촉 물체가 센서를 어떻게 누르고 있는지를 통합적으로 이해할 수 있습니다
    

## 6. Slip-Sensing을 DWF 또는 WF 방법 말고도 측정할 수 있게 하는 방법이 있는가?

- 챗지피티 답변 : 가능
    
    미끄럼 감지를 위해 BaroTac 연구팀은 **고주파 진동** 신호에 주목했습니다. 일반적으로 물체가 미끄러지면 센서 출력에 고주파 진동이 발생하므로, Fourier 변환(FT)이나 고역 통과 필터로 주파수 성분을 분석할 수 있습니다.  FT(고속 푸리에 변환)는 전통적으로 사용되나, 고정된 윈도우 분석이 필요해 실시간성에 제약이 있습니다. 이들의 연구에서는 대신 이산 웨이블릿 변환(DWT)을 선택하여 시간 도메인상에서 다수의 주파수 밴드를 추출했고, 75–150Hz 대역 성분 유무로 미끄럼을 판별했습니다. 웨이블릿 외에도 **단순 스펙트럼 분석** (예: STFT)이나 대역 통과 필터링, 고주파 값의 분산 계산 등으로 진동을 검출할 수 있습니다. 최근에는 딥러닝 기반의 방법도 제안되었습니다. 예를 들어 Grover 등(2022)은 **시계열 컨볼루션 신경망** (TCN)을 훈련시켜 바로메터 센서 신호로부터 미끄럼을 학습적으로 분류하였으며, 높은 정확도와 강인성을 보였습니다. 즉, FT/DWT 대신 시계열 데이터를 입력으로 하는 머신러닝(예: 1D CNN, RNN)으로 미끄럼/비미끄럼을 구분하는 것도 가능합니다.
    

## 7. 내가 구현하려는 3가지 기능을 모두 가능하게 하는 센서를 만들게 되면 고려되는 문제점이 무엇인가?
MCU의 스펙, Control 유닛, 통신속도, 배터리 소모 등

- 챗지피티 답변 : 가능
    
    세 가지 기능(3축 힘, 접촉 위치/분포, 미끄럼 감지)을 동시에 구현할 때는 하드웨어적·소프트웨어적 제약을 검토해야 합니다. 
    
    첫째, **MCU 성능**과 **통신 속도**입니다. Arduino DUE(84MHz) 같은 MCU로 16개 BMP384 센서를 SPI로 읽을 때, 샘플링 속도와 처리 부하를 고려해야 합니다. 예를 들어 미끄럼을 위해 100Hz 이상으로 데이터를 얻어야 하므로, 16채널을 동시에 높은 속도로 읽어올 수 있어야 합니다(BMP384는 SPI 10MHz까지 지원). SPI 버스 대역폭, 버스 상 멀티플렉싱 필요성, 읽기-처리 스케줄링 등이 중요하며, 인터럽트나 DMA를 적절히 활용해야 합니다.
    
    둘째, **실시간 처리 부담**입니다. DWT나 ML 예측 알고리즘은 계산 비용이 크므로 MCU에 부담이 될 수 있습니다. 필요한 경우, 미끄럼 감지 알고리즘은 비교적 간단한 필터로 구현하거나 별도의 DSP 칩/FPGA, 혹은 PC/모바일 디바이스로 연산을 오프로드할 수 있습니다. 힘 추정 및 위치 추정을 위한 ML 모델도 플로팅 연산이 많아, 모델을 경량화(소수 매개변수)하거나 단순화가 필요할 수 있습니다.
    
    셋째, **전력 소모**입니다. BMP384 센서는 절전 모드를 지원하지만, 샘플링 속도를 올리거나 오버샘플링률(해상도)을 높이면 전류 소비가 급증합니다. 예를 들어 BMP384는 저전력 모드(1Hz)에서 µA급 소비지만, 100Hz 이상에서 동작하거나 해상도 설정을 높이면 센서당 10µA~10mA 수준까지 소비할 수 있습니다(데이터시트 참조[bosch-sensortec.com](https://www.bosch-sensortec.com/media/boschsensortec/downloads/datasheets/bst-bmp384-ds003.pdf#:~:text=Forced%20Mode%20Max%20Ultra%20low,2)). 16개 센서를 연속 구동하면 상당한 전력이 요구되므로, 배터리 구동 시 전력 관리(읽기 주기 조절, 센서 클록 게이팅 등)가 필요합니다. Arduino DUE 등 MCU 소비도 포함하면, 전체 시스템 전력은 빠르게 증가합니다.
    
    넷째, **통신 논리와 동기화**입니다. 여러 센서를 읽고 필터링, 보정 후 통합하는 흐름을 구성해야 하며, 데이터 획득 시점(sync)이나 노이즈 차폐도 고려해야 합니다. 미끄럼 감지와 힘 측정이 동시에 이루어질 때는 우선순위 등을 명확히 설계해야 합니다.
    
    마지막으로, **환경 변화 대응**입니다. 바로미터 센서는 온·습도나 기계적 스트레인에 민감하므로, 온도 보상이나 센서 간 이질성 교정도 필수입니다. 또한 센서의 배열을 감싸는 엘라스토머 두께/경도 변화, 이음새 없는 접촉 표면 설계 등 기구적인 신뢰성을 확보해야 합니다.
    
    요약하자면, 이 세 기능을 한 장치에 통합하려면 **읽기/처리 속도**와 **전력 예산**, **계산 복잡도**를 면밀히 설계해야 하며, 필요한 경우 MCU나 센서 하드웨어 사양을 상향하거나, 계산을 분산하여 처리해야 합니다.

### 3-axis test Method 정리.

### Calibration Matrix Method

**데이터 테이블을 의사역행렬(pseudo-inverse)을 계산할 수 있는 행렬 형식으로 변환**해야 함. 

이는 센서 캘리브레이션에서 표준적인 과정으로, raw 데이터가 과잉 결정된 시스템을 다루기 위해 필요함. 연구에 따르면, 이러한 변환은 센서 간 편차를 보상하고 정확한 힘 추정을 가능하게 함.

**주요 단계 개요**

- 데이터 테이블(예: CSV나 스프레드시트)을 두 개의 주요 행렬로 추출: F (참조 힘 값, 3xN)와 S (센서 raw 값, 16xN).
- Baseline 보정을 적용해 드리프트를 제거.
- 의사역행렬을 사용해 캘리브레이션 매트릭스 C를 계산: C = F @ pinv(S').

**왜 변환해야 하는가?**
데이터 테이블은 일반적으로 시간, 위치, 힘 값, 센서 값 등이 섞인 행 기반 형식. 

하지만 **의사역행렬 계산**은 **선형 대수 연산을 요구**하므로, 이를 행렬(예: NumPy 배열)로 재구성해야 함.

이는 **과잉 결정된 방정식(입력 차원 > 출력 차원)을 풀기 위한 최소자승법의 일부임.**

**변환 방법 예시**
Python을 사용해 테이블을 불러온 후 행렬로 변환하세요. 예를 들어 pandas로 CSV를 읽고 NumPy로 변환:

```python
import pandas as pd
import numpy as np

df = pd.read_csv('your_data_table.csv')
F = df[['Fx', 'Fy', 'Fz']].values.T  # 3xN
S = df[['s1', 's2', ..., 's16']].values.T  # 16xN

```

**잠재적 문제와 팁**

- 데이터가 불완전하거나 노이즈가 많으면 condition number를 확인하고 정규화(예: Tikhonov)를 추가하세요.
- 다양한 포인트(P1~P9) 데이터를 포함해 일반화된 매트릭스를 얻으세요.

### 데이터 테이블의 일반적 구조와 문제점

사용자가 만든 데이터 테이블은 보통 다음과 같은 열을 포함합니다: Time, Pi (Sensing Point 라벨, 1~9), Fx, Fy, Fz (참조 힘, AFD50 센서에서), s1~s16 (BMP384 raw count). 이는 행 기반(각 행이 하나의 샘플)으로 저장되며, 총 N개의 샘플(포인트 × 하중 조건 × 반복)이 있습니다.

문제점:

- 이 형식은 선형 대수 연산에 직접 사용 불가능합니다. 의사역행렬은 행렬 곱셈을 요구하므로, 데이터를 **F (3xN, 참조 힘)와 S (16xN, 센서 값)로 재구성**해야 합니다.
- Baseline 드리프트(초기 오프셋)가 무시되면 오차가 증가합니다. BMP384는 온도 민감성(±1.9 Pa/K)이 있어 매 실험마다 보정이 필요합니다.
- 센서 간 감도 차이(제작 오차)로 인해 단순 평균이 아닌 최소자승 접근이 필수입니다.

### 데이터 준비 및 Baseline 보정

1. **테이블 불러오기와 필터링**: CSV나 Excel에서 데이터를 불러 Pi 라벨로 필터링하거나 전체를 사용하세요. 다양한 P_i 포인트 데이터를 포함하면 매트릭스의 일반화가 향상됩니다.
2. **Baseline 계산**: 무부하 상태 데이터(힘=0)에서 각 s_i의 평균 b_i를 구하세요. 이는 센서 드리프트를 제거합니다.
    - 예: b = 평균([s1, s2, ..., s16] over no-load samples)
3. **보정 적용**: 모든 샘플에 S' = S - b (broadcasting). 이는 BaroTac에서 암시적으로 사용된 방법으로, 선형성을 유지합니다.

| 단계 | 설명 | 예시 입력 | 예시 출력 |
| --- | --- | --- | --- |
| 테이블 불러오기 | pandas.read_csv() | 'data.csv' with columns: Time, Pi, Fx, Fy, Fz, s1~s16 | DataFrame (N rows) |
| Baseline | 평균 over no-load | s1 avg=7813952, etc. | b (1x16 vector) |
| 보정 | S' = S - b | Raw S (16xN) | Corrected S' (16xN) |

### 행렬 형식으로 변환

데이터를 다음과 같이 재구성

- F = [[Fx1, Fx2, ..., FxN], [Fy1, ..., FyN], [Fz1, ..., FzN]] (3xN)
- S' = [[s1_1, s1_2, ..., s1_N], [s2_1, ..., s2_N], ..., [s16_1, ..., s16_N]] (16xN)

Python pseudocode:

```python
import pandas as pd
import numpy as np

df = pd.read_csv('your_data_table.csv')
# No-load baseline (assume first 100 rows are no-load)
b = df.iloc[:100][['s1','s2',...,'s16']].mean().values  # (16,)
# Full data
F = df[['Fx','Fy','Fz']].values.T  # (3, N)
S_raw = df[['s1','s2',...,'s16']].values.T  # (16, N)
S_prime = S_raw - b[:, np.newaxis]  # Broadcast subtraction

```

이 변환은 과잉 결정 시스템을 다루기 위해 필요하며, Vision-based Tactile Sensor 연구에서 유사하게 사용됩니다.

### 의사역행렬 계산

캘리브레이션 매트릭스 C (3x16)

$$
C = F \cdot (S'^T S')^{-1} S'^T = F \cdot \text{pinv}(S')
$$

- pinv는 Moore-Penrose 의사역행렬로, 입력 > 출력 차원에서 최소자승 해를 제공
- BaroTac에서: C = (S^T S)^{-1} S^T F (전치된 형태, 동일 결과).

Python 구현:

```python
C = np.dot(F, np.linalg.pinv(S_prime))

```

| 매트릭스 | 크기 | 역할 | 예시 값 범위 |
| --- | --- | --- | --- |
| F | 3xN | 참조 힘 | Fx: -3~3N, Fz: 0~5N |
| S' | 16xN | 보정 센서 | 0~7813952 (raw count) |
| C | 3x16 | 변환 계수 | c_ij: N/count 단위 |

### 검증 및 개선

- 테스트 데이터로  $\hat{F} = C S'_{test}$ 계산 후 RMSE/MAE 평가.
- 문제: Condition number 크면 Tikhonov 정규화 (pinv(S' + λI)).
- Superresolution 연구처럼 isolines 이론 적용으로 정확도 향상 가능.
- 추가: 온도 보상(Temp 열 사용)이나 비선형 모델(MLP)로 확장.

이 과정은 iFEM2.0과 같은 고급 방법으로 확장 가능하며, 접촉력 재구성에 유용합니다.
