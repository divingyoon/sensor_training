# 센서 receptive field 시각화 (방법 설명)

> 생성: `python visualizing_scripts/xy_1mm/generate_receptive_field.py --diameter all`
> 출력: `fig2_heatmap/hitmap/` — 센서별 `<dia>_<material>_sensors.png`(6), 겹침 `overlap_<dia>_{2d,3d}.png`(4),
> 2번째센서 `second_sensor_<dia>.png`(2), 다중센서 coverage `coverage_<dia>.png`(2),
> 인덴터 크기별 `spreading_gain.png`(1), 수치 `STATS_수치.md`.

## 그림의 두 역할 (중요)
이 산출물들은 **목적이 둘로 갈린다**. 혼동 금지:

| 그림 | 역할 | 답하는 질문 |
|---|---|---|
| **overlap** `overlap_<dia>` + 센서별 `<dia>_<mat>_sensors` | **① 센서 설계 근거(receptive)** | 수용장이 인접 taxel 과 겹치게 만들어졌나 → **왜 이 소재/구조로 제작?** |
| **second_sensor** `second_sensor_<dia>` + **coverage** `coverage_<dia>` + `spreading_gain` | **② SATS 학습 데이터 품질** | 실제 녹화한 데이터가 SR 학습에 쓸 다중센서 신호를 담고 있나 → **이 데이터로 SATS 학습이 되나/좋은 상태인가?** |

- **① overlap = 설계/제작 정당화**: O_excess 로 "수용장이 서로 겹친다"는 **센서 만든 의도**를 보임(receptive field 관점, 절대크기 무관 정규화 겹침 기하). = mesh/eco50 를 고른 이유.
- **② coverage·second_sensor = 학습데이터 검증**: SATS 입력은 ΔS%(floor 없음) 이므로, **녹화 데이터 자체**가 각 위치에서 ≥3 센서(또는 강한 2번째 센서) 신호를 담으면 **SR 학습이 잘 된다는 증거**. = "이 d5/d10 데이터는 SATS 학습용으로 충분/양호".

## 핵심 정의
각 센서 s_i 가 **인덴터를 (x,y) 에 최대깊이(z_max)로 눌렀을 때 얼마나 반응했는지**(|ΔS_i|, %)
를 (x,y) 히트맵으로 = 그 센서의 **수용장(receptive field)**. 신호 = per-press local baseline ΔS,
**접촉 구간 z≥13mm 의 셀별 최대(z_max 압입) |ΔS|** (travel/대각선 이동 구간 제외 → 아티팩트 차단).

> **d5 와 d10 의 겹침 격자 차이**: overlap 의 격자 = 인접 센서 수용장이 겹쳐 그 사이(2-way 중점)가
> 밝아진 것. **d10(⌀10mm > 센서간격 6.5mm)** 은 한 압입이 여러 센서에 넓게 걸쳐 **강한 겹침 격자**가 생긴다
> (강한겹침 O≥0.3 면적 175~243mm²). **d5(⌀5mm < 6.5mm)** 는 국소 압입이라 수용장이 좁아 겹침이 약하다 —
> 디노이즈 후 **희미한 격자**는 보이나(중간겹침 O≥0.2 면적 0~69mm², 강한겹침 ≥0.3 은 0), d10 의 ~3배 약하다.
> 즉 **겹침(SR 입력) 시연의 본 무대는 d10**, d5 는 좁은 수용장의 한계를 보여주는 대조군.

## d5 디노이즈 (약신호 speckle 제거)
- d5 는 인덴터가 작아(⌀5mm) 응답이 약하고 셀별 speckle 노이즈가 커 보인다. 처리:
  - **몽타주(센서별)**: 가우시안 블러(σ0.8) + 자기 peak 30% 미만을 0 으로(상대 노이즈플로어) → 봉우리만 남김.
  - **overlap**: 블러(σ0.9)만 적용(하드 floor 없음). 하드 floor 를 정규화 전에 걸면 좁은 d5 수용장이
    겹침 0 이 되어버리므로, 약하지만 실재하는 겹침을 보존하려 speckle 만 평활.
- mesh d5 가 "무늬 없이 지저분"해 보이던 원인 = 센서별 peak 편차가 큼(30~47%)이라 밝기가 들쭉날쭉.
  eco20 은 peak 가 균일(15~19%)해 매끈해 보였을 뿐 신호는 더 약함. 디노이즈+정규화 후 mesh 도 격자 겹침이 또렷.
- d10(강신호)에는 디노이즈 미적용(원시 신호로 충분).

## A. 센서별 receptive field — `<dia>_<material>_sensors.png`
- 4×4 몽타주(센서 물리 배치). 색(turbo) = |ΔS| 응답크기. 자기 위치(+)에 봉우리, 멀수록 감소.
- **색 스케일은 eco20/eco50/mesh 3소재 공통(vmax 동일)** → 같은 기준으로 직접 비교 가능.
  d5 에서 eco20 은 peak ~17% 로 어둡게(약함), eco50·mesh 는 ~40%·37% 로 밝게 나온다(16채널 평균, STATS A절).

## B. 다중센서 겹침 overlap — `overlap_<dia>_{2d,3d}.png`
- `O_excess(x,y) = Σ_t(|ΔS_t|/peak_t) − max_t(|ΔS_t|/peak_t)`
  = 각 센서를 자기 peak 로 정규화해 더한 값에서 **가장 센 센서 1개를 뺀 '추가 중첩분'**.
- 의도: **센서 자기 중심(혼자 dominant)은 빼서 어둡게** 되고, **여러 수용장이 겹쳐 여러 센서가
  동시에 반응하는 영역일수록 밝게** → 그림의 밝은 곳 = 센서 위치가 아니라 **센서 사이 겹침대**.
  절대 응답크기와 무관 → 실제 ΔS 총합이 낮아도 겹침이 높게 표현(요청 의도).
- 2D + 3D(가우시안 스무딩). **밝은 곳 = 다중센서 동시반응 = SATS 가 x,y,z 보간에 가장 유리.**
- **컬러스케일은 d5·d10 공통(vmax≈0.74)** → overlap_d5 와 overlap_d10 그림을 직접 비교 가능.
  그 결과 d5 는 전체적으로 흐린 파랑(최대 0.30 ≈ d10 0.74 의 40%)으로 보이는데, 표시 오류가 아니라
  **d5 겹침이 d10 보다 ~3배 약하다**는 사실의 반영이다.
- **겹침면적**: 공유 스케일 기준 **중간 O≥0.2 / 강한 O≥0.3** 면적으로 측정(STATS B절).
  이전 O>0.05 는 거의 전 격자(441mm²)가 잡혀 포화→변별 불가였음. 0.2/0.3 에서 소재가 또렷이 갈린다.
- **읽기(d10)**: 겹침 평균 eco20 0.147 ≪ mesh 0.300 < **eco50 0.351**(p95 0.30 ≪ 0.55 < 0.64),
  강한겹침(O≥0.3) 면적 **eco50 243 > mesh 175 ≫ eco20 21 mm²**(eco20 의 8~12배).
  → eco50·mesh 는 센서 사이가 밝은 그물망(넓은 다중겹침), eco20 은 파랑(겹침 빈약).
  **mesh 와 eco50 는 패턴은 같으나 eco50 가 ~17% 더 진하다(동일 아님)** — 그림: eco50 빨강, mesh 주황.

## C·D. SATS 학습신호 — 순위별 센서 맵  `second_sensor_<dia>.png`(2번째) · `coverage_<dia>.png`(3번째)
**두 그림은 같은 연속 형태** → 직접 비교. 각 위치에서 **k번째로 센 센서의 |ΔS|** 를 연속 색으로(임의 floor 없음).
근거: **SATS 입력 = ΔS%, floor 없음**(`dataset.py:248`) → 1~2% 약신호도 그대로 학습에 씀. 정량: 패널마다
**중앙값 + (그 센서 ≥1% 인 면적%)**. 색 스케일은 **각 그림(rank)의 3소재 공통**(자기 스케일) → 소재끼리 비교되며
d5·d10 모두 또렷(2번째 vmax 가 3번째를 깔아뭉개지 않게 rank 별로 분리). 표시는 업샘플+스무딩으로 매끈한 연속 heatmap.
- **C. 2번째 센서**(SR 최소 ≥2): d5 eco20 중앙값 **0.5%(≥1% 면적 5%)=노이즈급** → 단일 센서. eco50 **1.8%(98%)**·mesh **1.7%(96%)** → 이웃 신호 넓게.
- **D. 3번째 센서**(모호함 없는 SR 바 ≥3; 2D 위치+미지 접촉력=미지수 3 → 3 센서 필요): d5 eco20 **0.3%(0%)** → SR 불가.
  eco50 **1.2%(76%)**·mesh **1.2%(68%)** → 면적 대부분 3번째 센서가 노이즈 위 = SR 학습 가능.
- **왜 이 형태**: "센서 수 N(이산)"은 floor 근처에서 셀마다 튀어 second_sensor 와 형태가 안 맞음. **순위별 |ΔS|(연속)**
  으로 통일해 2번째↔3번째 직접 비교 + 정량. SATS 구조도 16채널 self-attention+local map 합산 → 반응 센서 모두 통합.
- d5 는 약블러(σ0.6)로 셀 speckle 만 평활(약신호 inflation 최소). d10 은 raw. coverage_stats 의 ≥3 카운트와 동일 소스.
- ⚠ 이전 hard-count(floor 3%) 버전은 3번째 센서(~1.2%)를 잘라 ≥3 가 거짓 0~4% 로 보였음 → floor 노이즈급(1%)·연속 형태로 교정.

## E. 인덴터 크기별 추세 — `spreading_gain.png`
- ≥3센서 coverage(floor 1.0%)·3번째센서 중앙값·overlap 평균을 **d10↔d5** 로 이어 그림.
- **d10**: 인덴터가 커 세 소재 모두 다수 센서 자극 → 소재차 작음. **d5**: **eco20 만 붕괴**(3번째센서 0.2% 노이즈,
  ≥3 coverage 0%), eco50·mesh 는 ≥3 센서 SR 신호 유지 → **압력분산 이점은 접촉이 작을수록(=d5, SATS 대상) 극대화**.

## 3. eco50 d5 채널 보충(merge)
- eco50 d5 는 완전한 16채널 녹화가 없음: test1(s16 stuck), test2(s1·s2 rail), test3(s1 rail).
- 처리: **test3 를 기본**으로 쓰되, test3 에서 죽은 **s1 은 test1(거기선 정상)의 수용장으로 보충** →
  **16채널 모두 생존**(센서별 수용장은 각자 독립적으로 구성되므로 채널별 다른 녹화 결합이 타당).
- 영구 고장 아님(다른 녹화/인덴터는 전 채널 정상).

> 이전 onset-깊이 hitmap·max-envelope·단순합(sum) 버전은 폐기. 본 (응답크기 + 정규화 겹침) 방식이
> "센서별 수용장 + 수용장 중첩"을 의도대로 보여준다.
