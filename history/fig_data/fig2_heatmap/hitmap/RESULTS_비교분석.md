# 센서 receptive field 결과 비교분석

> 대상: `fig2_heatmap/hitmap/` — 센서별 `<dia>_<mat>_sensors.png`(6), 겹침 `overlap_<dia>_{2d,3d}.png`(4).
> 신호 = per-press local baseline ΔS, 셀별 z_max(최대압입) |ΔS|. eco50 d5 = test3 + s1(test1 보충) → 16채널 생존.
> **수치 전체(센서별 수용장 넓이·overlap 분포 통계)는 [`STATS_수치.md`](STATS_수치.md) 참조.**
> d5(약신호)는 디노이즈(가우시안 σ + 자기 peak 상대 노이즈플로어) 후 그림/통계 산출.

## 0. 정량 요약

**(a) 센서별 수용장 넓이** (16채널 평균; peak=|ΔS|%, half-max 면적=자기 peak 50% 이상 셀):

| 인덴터·소재 | peak(%) | half-max 면적(mm²) | 등가지름(mm) |
|---|---|---|---|
| d5 eco20 | 17 | 8.4 | 3.2 |
| d5 eco50 | 40 | 9.2 | 3.4 |
| d5 mesh | 37 | 9.6 | 3.5 |
| d10 eco20 | 99 | 14.1 | 4.2 |
| d10 **eco50** | 100 | **23.2** | **5.4** |
| d10 mesh | 100 | 21.4 | 5.2 |

> 수용장 넓이: **eco50 ≳ mesh ≫ eco20**. d10 에서 eco50·mesh 수용장이 eco20 보다 ~1.6배 넓다(피치 6.5mm 에 근접 → 이웃까지 닿아 겹침 생성). eco20 은 좁아 undersampling.

**(b) 다중센서 겹침 O_excess=Σ(|ΔS_t|/peak_t)−최강** — 센서 중심은 빼고 **겹침 영역만** 높게:

| 인덴터·소재 | 평균 | CV | p95 | 중간겹침 O≥0.2(mm²) | 강한겹침 O≥0.3(mm²) |
|---|---|---|---|---|---|
| d10 eco20 | 0.147 | 0.53 | 0.30 | 108 | 21 |
| d10 **eco50** | **0.351** | 0.41 | **0.64** | **397** | **243** |
| d10 mesh | 0.300 | 0.43 | 0.55 | 342 | 175 |
| d5 eco20 | 0.098 | 0.29 | 0.15 | 0 | 0 |
| d5 eco50 | 0.139 | 0.24 | 0.20 | 19 | 0 |
| d5 **mesh** | **0.157** | 0.28 | 0.24 | **69** | 0 |

> **겹침면적**: 공유 스케일(vmax 0.74) 기준 임계값. 이전 O>0.05 는 거의 전 격자(441mm²)가 잡혀 **포화→변별 불가**였음.
> O≥0.2/0.3 으로 보면 **d10 강한겹침 eco50 243 > mesh 175 ≫ eco20 21 mm²**(8~12배 차)로 또렷이 갈린다.

> **mesh ≠ eco50 (동일 아님)**: d10 겹침은 **eco50(0.351) > mesh(0.300)** 로 eco50 가 ~17% 더 강하다(p95 0.64 vs 0.55).
> 그림에서 eco50 가 더 진한 빨강, mesh 는 주황으로 보이는 것과 일치. 패턴(격자 위치)은 같지만 **세기가 다르다**.
> d5 는 디노이즈 후 mesh(0.157) ≳ eco50(0.139) > eco20(0.098) — d5 에서는 mesh 가 근소 우위.

## 1. 센서별 receptive field (`<dia>_<mat>_sensors.png`)

각 칸 = s_i 가 z_max 압입에 반응한 크기. 색 스케일 공통(직접 비교 가능).

- **d10**: 세 소재 모두 peak ~100% 포화. 봉우리 **폭**이 차이 — eco50·mesh 가 더 넓게 퍼지고, mesh 가 가장 원형·등방.
- **d5**: peak 자체가 **eco20 22% ≪ eco50 47%·mesh 44%** → eco20 은 어둡게(약함) 표시. 봉우리도 좁다.
- **eco50 d5 의 s1(-9.75,-9.75)**: test3 에서 dead 였으나 **test1 에서 보충**해 정상(peak 47%) 표시(§3).

## 2. 겹침 overlap (`overlap_<dia>_{2d,3d}.png`)

`O_excess = Σ(센서/자기peak) − 최강센서` — 센서 자기중심은 빼고 **다중센서 겹침 영역만** 밝게. 절대크기 무관.

> **overlap_d5_2d 와 overlap_d10_2d 는 컬러바(vmax=0.74)가 동일** → 두 그림을 직접 비교 가능.
> 그 결과 **d5 는 전체적으로 흐린 파랑**(최대 0.30 ≈ d10 의 0.74 의 40%)으로 보이는데, 이는 표시 오류가 아니라
> d5 수용장(⌀~3.4mm < 피치 6.5mm)이 좁아 **겹침 자체가 d10 보다 ~3배 약하다**는 사실의 정직한 반영이다.
> (d5 소재 간 미세 차이를 보려면 STATS B절의 수치를, 시각적으로는 흐리게 보이는 게 정상.)

### d10 (결정적)
- **겹침 평균: eco20 0.147 ≪ mesh 0.300 < eco50 0.351**, p95 도 eco50 0.64 > mesh 0.55 ≫ eco20 0.30.
- **mesh 와 eco50 는 격자 패턴(겹침 위치)은 같으나 세기가 다르다** — eco50 가 ~17% 더 진하다(그림: eco50 빨강, mesh 주황). "동일"이 아니라 "**같은 계열, eco50 가 근소 우위**".
- 같은 위치라도 센서 중심은 더 어둡다 → 그림의 밝은 곳 = 센서 위치가 아니라 겹침대 = SATS 가 보간할 단서가 풍부한 영역.
- 3D: eco50·mesh 는 센서 사이에 봉우리(겹침), eco20 은 거의 평평·낮음.

### d5 (디노이즈 후)
- 원시 d5 는 셀별 speckle 가 커 보였음 → **가우시안 블러 + 자기 peak 상대 노이즈플로어**로 평활.
- 디노이즈 후 격자 패턴이 드러남: 평균 **mesh 0.157 ≳ eco50 0.139 > eco20 0.098**, CV 0.24~0.29(평활됨).
- **mesh 가 "무늬가 없다"던 인상의 원인**: mesh d5 는 센서별 peak 편차가 큼(30~47%, [`STATS_수치.md`](STATS_수치.md) C절) → 정규화 전엔 밝기 들쭉날쭉. eco20 은 peak 가 균일(15~19%)해 매끈해 보였을 뿐 신호는 더 약함. 디노이즈+정규화 후엔 mesh 도 또렷한 격자 겹침을 보인다.
- 단, d5 수용장(⌀~3.4mm) < 피치 6.5mm 라 겹침 절대량은 d10 보다 작다 → **겹침 시연의 본 무대는 d10**.

## 3. eco50 d5 채널 보충
- 완전한 16채널 녹화 없음(test1 s16 stuck / test2 s1·s2 / test3 s1) → **test3 기본 + s1 만 test1 에서 보충** = 16채널 생존. 영구 고장 아님. 코너 채널이라 결론 영향 없음.

## 3b. SATS 학습 데이터 품질 — d5 다중센서 신호 (≠ 설계용 overlap)

> **역할 구분**: §1~3 의 **overlap = 센서 설계 근거**(수용장이 겹치게 제작했나, receptive 관점).
> 아래 **coverage·second_sensor = 학습 데이터 품질**(녹화한 d5/d10 이 SR 학습에 쓸 다중센서 신호를 담았나 = "이 데이터로 SATS 학습 가능/양호").


> `second_sensor_<dia>.png`, `coverage_<dia>.png`, `spreading_gain.png`. SATS 입력 = ΔS%(**floor 없음**,
> `sats/training/dataset.py:248`) → 네트워크는 1~2% 약한 2번째 센서 신호도 그대로 씀. (수치 STATS B2절)
> ⚠ 이전 floor 3% 는 이 신호를 잘라 ≥2 영역이 거짓 0% 로 보였음 → **floor 를 노이즈급(1%)으로 교정**.

> **왜 ≥3 인가**: 2D 위치(x,y)+미지 접촉력 = 미지수 3 → 모호함 없는 SR 은 **≥3 센서** 필요(2 는 거리-링 2교점 모호).
> SATS 도 16채널 self-attention + 센서별 local map 합산 + CNN refiner → **반응하는 센서를 모두 통합**(2-센서 삼각측량기 아님).

- **시각화 형태(중요)**: `second_sensor`(2번째 센서)와 `coverage`(3번째 센서) 는 **동일 연속 형태·공통 colorscale** —
  각 위치 k번째로 센 센서의 |ΔS| 를 연속 색으로(흰 점선=1% 노이즈 경계). 정량(중앙값+≥1% 면적%)·시각화 동시. 직접 비교됨.
  (이전 '센서 수(이산)' 맵은 floor 근처에서 셀마다 튀어 second_sensor 와 형태가 안 맞았음 → 순위별 |ΔS| 연속으로 교체.)
- **2번째 센서(SR 최소 ≥2)**: d5 **eco20 중앙값 0.5%(≥1% 면적 5%)=노이즈급 ≪ eco50 1.8%(98%) ≈ mesh 1.7%(96%)**.
- **3번째 센서(모호함 없는 SR 바 ≥3)**: d5 **eco20 0.3%(0%) ≪ eco50 1.2%(76%) ≈ mesh 1.2%(68%)**.
  eco20 은 2·3번째 센서가 노이즈급 → **단일 센서만 반응 = SR 학습 불가**. eco50·mesh 는 면적 대부분에서 3번째 센서까지 노이즈 위 = **SR 학습 가능/양호**.
- **spreading_gain(d10↔d5)**: d10 은 셋 다 다수센서 자극(소재차 작음). **d5 에서 eco20 만 ≥3 coverage 0% 로 붕괴**,
  eco50·mesh 유지 → **압력분산 이점은 접촉이 작을수록(=d5) 극대화**.
- **mesh vs eco50**: d5 다중센서 coverage 는 둘이 막상막하(eco50 근소 위), mesh 는 보조센서 최대(2nd 5.1%)·정규화 overlap(0.157) 우위.
  **핵심 결론 = eco20 의 완전 실패(≥3 불가) 대비 eco50·mesh 가 ≥3 센서 SR 신호를 넓게 만든다**(SATS 를 d5 로 하는 근거).
- ⚠ 3번째 센서가 ~1.1% 로 floor 근방이라 ≥3 coverage 는 floor 민감(1.0%→66/56%) → 견고한 비교는 **3rd-센서 세기 자체**(1.1% vs 0.2%, 5배).

## 3c. ecomesh 의 불규칙한 패턴 — 원인과 학습 관점

`second_sensor_d5`·`coverage_d5` 에서 eco50 은 깔끔한 격자, **mesh 만 blotchy(불규칙)** 하게 보이는 것에 대한 분석.

**원인 단계 배제(모두 측정):**
- 정렬: peak 위치 offset eco50·mesh 둘 다 <1mm (정상).
- 수용장 폭: width CV 둘 다 0.13 (동일).
- peak gain: **per-sensor 정규화(각 센서 /자기 peak)해도 mesh 여전히 불규칙** → gain 만의 문제 아님.
- 남은 원인 = **수용장의 약한 꼬리(2·3번째 센서가 잡는 ~1% 영역)가 taxel 마다 이방성·불균일하게 퍼짐** + per-taxel peak gain 무작위(CV 0.14 vs eco50 0.05).

**물리적·재현가능 (결정적):** mesh d5 **test1/2/3 (3회 독립 녹화)의 4×4 peak 지도 상관 = 1.00** — 같은 taxel 이 항상
약하고(예 34·36%) 같은 게 강하다(58/57%). → **녹화 노이즈·정렬 오류·viz 버그가 아니라 제작된 그물층의 taxel 별
고정 결합 세기**(이산 interconnect 구조 각인). eco50(균질 Ecoflex)은 등방·매끈하게 퍼져 규칙적.

**학습에 유리한가? — "무해, 조건부 소폭 유리"(과신 금지):**
- **핵심 = 재현가능(corr 1.00)** 하므로 **고정 calibration** 으로 네트워크가 1회 학습으로 흡수 → **무해**.
  (만약 매번 다른 *랜덤* 노이즈였다면 학습 불가 = 치명적. 재현가능이 핵심.)
- **소폭 유리 가능**: 채널별 응답 다양성이 forward map 에 per-channel 특이성(고정 지문)을 더해 위치 추론 단서가 될 수 있고,
  서로 다른 gain 이 힘 구간별로 채널을 민감영역에 분산(dynamic-range 다양성). = 데이터가 더 '구별 가능'.
- **소폭 불리 가능**: 약한 taxel(34%)은 강한 것(58%)보다 SNR 낮음.
- **종합**: net 무해~약간 유리, 절대 불리는 아님. 단 "설계된 강점"으로 과대포장은 금물 — 주된 SR 동력은 여전히 overlap/coverage.

## 4. 종합 결론

1. **eco20 = 일관 최저** — 수용장 좁고(d10 면적 14mm²), 겹침 O 최저(d10 평균 0.147) = undersampling.
2. **겹침(overlap)은 eco50 최강·mesh 근소 차 2위, 둘 다 eco20 보다 월등** (d10 평균 0.351 / 0.300 vs 0.147).
   SR/SATS 가 (x,y,z) 를 추론하려면 여러 센서가 동시에 반응해야 하는데, eco50·mesh 가 그 **다중-센서 중첩 영역이 넓다**.
3. **mesh 의 고유 강점** = "겹침 1등"이 아니라 **eco50 에 근접한 겹침/coverage(겹침 0.300 vs 0.351, 약 85%)를 유지하면서**
   **가장 원형·균일한 수용장 + 민감도·active taxel 수(Fig2C 최대)를 함께 확보** → eco20 의 undersampling 도, (필요시) eco50 의 포화/번짐도 피하는 균형. (정직한 C1)
4. **mesh 와 eco50 는 동일하지 않다**: 겹침 세기는 eco50 가 근소 우위(~17%). 두 소재가 그림상 다르게 보이는 것은 정상이며, 데이터와 일치한다.
5. **읽기 주의**: d10 은 peak 포화 → **겹침 O·수용장 면적**으로 본다. d5 는 약신호 → 디노이즈 후 상대 패턴.
