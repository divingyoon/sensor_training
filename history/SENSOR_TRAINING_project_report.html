1. 프로젝트 목표

핵심 문제

실제 촉각 센서는 4 x 4, 총 16개의 sparse sensor만 가지고 있습니다. 목표는 이 제한된 센서 응답으로 전체 표면의 dense pressure map을 추론하는 것입니다. 현재 기본 출력은 [-10, 10] mm 영역을 0.5 mm 간격으로 표현한 41 x 41 virtual taxel map입니다.

논문 기반 방향

참고 논문 Super-resolution tactile sensor arrays with sparse units enabled by deep learning은 적은 수의 physical taxel 신호를 LSTM, self-attention, local map, CNN으로 처리해 훨씬 촘촘한 virtual pressure map을 생성합니다. 본 프로젝트는 이 구조를 mk555 장비와 현재 raw BIN 데이터 구조에 맞게 재구성합니다.

2. 데이터 수집 구조

각 trial은 동일한 timestamp suffix를 가진 세 종류의 raw BIN으로 구성됩니다. EtherMotion은 고속 stage command/log를 제공하고, DUE는 16채널 센서 값을, loadcell은 수직 힘을 제공합니다.

Source 파일 역할 학습에 들어가는 값
DUE due_raw_burst_*.bin 16개 tactile sensor stream. 1 burst에 FIFO 10 frame 포함. s1..s16, baseline normalization 후 model input
EtherMotion ethermotion_encoder_*.bin 고속 stage command/log. 현재 1000Hz 이상으로 기록됨. x_mm, y_mm, z_stage_mm, z_depth_mm, u_mm
Loadcell loadcell_raw_*.bin 단일축 loadcell kg stream. Fz = (kg - kg_baseline) * 9.80665
d5 기준으로 stage는 z_stage_mm = 13.0에서 시작하고 15.5까지 내려갑니다. 따라서 전처리에서는 z_depth_mm = max(z_stage_mm - 13.0, 0)으로 해석합니다. u_mm은 0.5, 1.0, 1.5, 2.0mm 등에서 잠깐 멈추기 위한 가상축이며 물리적 shear 축으로 쓰지 않습니다.

4. Ground Truth 생성 전략

기존 방식: precomputed GT

모든 200Hz merged row마다 41 x 41 float32 pressure map을 미리 계산해 learning_data/gt/*_targets.npy로 저장합니다. d5 한 trial은 약 2.74M row라서 GT 파일 하나가 약 18GB까지 커집니다.

  • 장점: 학습 중 GT 계산 비용이 작음
  • 단점: 10개 trial이면 GT만 약 180GB 이상 필요
  • 기존 코드와 실험 호환성 때문에 유지

신규 방식: on-the-fly GT

dense GT 파일을 저장하지 않고, 학습 batch를 만들 때 merged row의 x/y/z_depth/Fz로 pressure map을 즉석 생성합니다. 같은 접촉 반경과 위치에 대한 Boussinesq kernel은 cache합니다.

  • z_s_mm = 2.0 fixed
  • FEM beta correction 없음
  • 구형 인덴터: a = sqrt(R * z_depth)
  • d5 접촉 시작: z_depth > 0.001mm
현재 0.005mm는 z-depth 샘플링/균형화 단위입니다. XY pressure map의 공간 간격은 현재 0.5mm입니다. XY virtual taxel을 더 촘촘하게 하려면 model/GT output grid를 81 x 81 같은 구조로 키워야 합니다.

5. 논문 SATS 구조와 현재 모델

논문은 physical taxel 수보다 훨씬 많은 virtual taxel pressure map을 출력합니다. 본 프로젝트도 같은 철학을 따릅니다. 16개의 sparse sensor response를 시간 특징으로 인코딩하고, sensor 간 attention으로 공간 정보를 교환한 뒤, local map과 CNN으로 전체 pressure map을 복원합니다.

6. 학습 샘플링 정책

U 축 정지 시간이 trial마다 다르면 plateau row 수가 달라져 특정 깊이 구간이 과대표집됩니다. 따라서 모든 200Hz row를 같은 가중치로 쓰지 않고, contact phase를 균형 있게 샘플링합니다.

Dynamic / Loading

z-depth와 Fz가 증가하는 구간입니다. 센서의 동적 변화, 지연, hysteresis를 학습하는 핵심 구간입니다.

Plateau / Static

stage가 멈춰 있고 압력 분포가 유지되는 구간입니다. 정지 압력맵을 지속적으로 추론하기 위해 일부 보존합니다.

Saturation / Peak

최대 깊이와 높은 Fz 주변 구간입니다. 큰 압력과 센서 포화 응답을 놓치지 않도록 보존합니다.
